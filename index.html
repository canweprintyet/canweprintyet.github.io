Can we print yet?

No. :-(

Overview

In 1439, Gutenberg's printing press began allowing for mass production of documents.
In the early 1900s, offset lithography started to help make newspapers and magazines with good-fidelity color images widely available.
It is 2015, and printing webpages in major modern Web browsers is a fairly bug-ridden affair.

And we're not even talking about "fancy" "cutting-edge" stuff like the CSS Paged Media Module Level 3, much less the Page Floats or GCPM CSS specs. We're talking about more basic stuff, like actually physically printing elements at the exact size you explicitly declared them to be!

It shouldn't be like this. Since pretty early on, CSS was designed to take into account the need to print things. CSS2 defined a print media type and @page rules. CSS1 defined inch, cm, mm, and point units. When combined with min-width and max-width media queries, these together should have been sufficient to allow Web designers to achieve a great many things with print media with relative ease. Instead, they are left to wade through a thicket of bugs. Even in an era when traditional print media is on the decline, there are still many cases where hard copies are necessary; Amazon has to put shipping labels on boxes somehow. And so today's best-practice has become to generate PDFs or raster images, and give up on serious usage of the Web for print media.

With the Web, you might be able to process data with transactional safety, play an online 3D videogame while video-chatting with your teammates, or show your friend a funny cat video and then find nearby hospitals when they start dying of laughter, but heaven forbid you should want your online article to look nice when someone prints it out to show it to their grandfather...

The Bugs

We'll now proceed to detail the bugs that currently make the life of print stylesheet designer very painful.

Chrome incorrectly sizes elements when printing (Bug #105201)

This is by far the most egregious bug. Due to Chromium bug #105201, which has been open since 2011, Chrome prints elements slightly larger than their explicit CSS sizes (even when print margins are disabled, even when the sizes are declared within a @media print media query).

Call to action

We call upon the browser vendors to stop overlooking these languishing printing-related bugs and start prioritizing the fixing of them.

We call upon fellow Web designers and Web developers everywhere to (depending on the bug tracking system in use) star or vote for or add themselves to the CC lists of the browser bugs listed below, to let the browser vendors know that we care about getting them fixed.

And of course, it would be wonderful if there was greater progress in the testing and standardization of printing on the Web, but we won't hold our breath.
